# 写作理论内核（L0 唯一权威源）

本文件是整个技能包的公理层。所有创作规则（L1）、风格指纹（L2）、开书策略（L3）、章级追读（L4）、长线资产（L5）、质量治理（L6）和数据复盘（L7）都必须能回溯到这里的五个维度。回溯不到的规则，要么删掉，要么是错的。

## 一、核心命题

一部适合番茄中文网长篇连载的小说，必须同时满足五个维度：像人写、有节奏、有信息、有追读、有平台留存。写得好、减少机器腔、让读者追读和适配推荐数据，本质上是同一条生产线的不同检查口。

## 二、五维模型

### 1. 突发性 Burstiness（B = σ_L / μ_L）

句与句、段与段的长度要参差。

- 机器腔的病：句长方差趋近于零，输出均匀。
- 人的特征：长短交错，参差不齐。
- 派生律：长短交错律（见 L1）。它同时说明——"全短句"和"全长句"都错，只有"长短交错"对。

### 2. 困惑度 Perplexity（PXP）

词链要不可预测。

- 机器腔的病：总走最高概率的词链（然而/因此/不禁/微微）。
- 人的特征：下一个词的选择概率分布宽，常有"意外"。
- 派生律：去高概率词链律（见 L1）。

### 3. 信息熵 Entropy（H(X)）

词汇、句首、句式要丰富。

- 机器腔的病：词汇贫乏、句首单一、句式重复。
- 人的特征：用词杂、开头多变、结构不雷同。
- 派生律：多样性律（见 L1）。

### 4. 追读力 Engagement（E）【新增】

**这段文字是否让读者想翻下一页？**

这是 wenxin-v6 新增的第四个维度。前三维解决"像人写的"，这一维解决"让人想读"。

- 机器腔的病：信息推进过匀，每段都是"动作+感受+总结"，没有悬念缺口。
- 人的特征：故意留白、延迟满足、制造悬念缺口。读者知道一部分，但不知道全部。
- 派生律：
  - 每 500 字至少有一个信息增量（新设定/新冲突/新反转/新关系/新发现）。
  - 章末必须有悬念缺口（新威胁/新能力/反常事件/未解之谜）。
  - 主角必须有主动性（不能只是被动应对）。
  - 每章至少有一个爽点时刻（打脸/升级/反杀/被认可/捡漏/扮猪吃虎）。
  - 配角反应必须有身份/性格区分（不能千人一面）。

**追读力计算公式（与 naturalness_scan.py 实现完全一致）：**

```
E = clamp(信息增量得分 × 0.3 + 章末钩子得分 × 0.3 + 主角主动性得分 × 0.2 + 爽点密度得分 × 0.2)
```

各分量计算方式（与脚本 naturalness_scan.py 完全一致）：

- **信息增量得分** = min(10, 信息增量密度 × 2)，满分 10
  - 信息增量密度 = 每 500 字的信息增量数
- **章末钩子得分** = 字母等级映射：S=10, A=8, B=6, C=3, D=1
- **主角主动性得分** = min(10, 选择类主动模式 ×2 + 行动类模式 ×0.5)，满分 10
  - 选择类主动模式：决定、选择、拒绝、答应、承认、揭开、锁定、反击、拦住、挡住、追上、抢先、主动、不等对方开口、没有等
  - 行动类模式：冲、挡、追、拦、躲、砸、踹、抓、按、扣、拔、掀、翻、递、收、藏、扔、抢、夺、转身、抬手、开口、推开、打开、关上、放下、拿起
- **爽点密度得分** = min(10, 爽点词组/结构命中 ×1.5 + 完整结构奖励)，满分 10
  - 爽点检测使用词组和结构模式，不使用“打/杀”等单字计数；优先识别打脸、反杀、突破、升级、被认可、捡漏、扮猪吃虎、揭穿、反转、惩罚、赔偿、奖励、压制、逆转、后悔，以及“压抑 -> 爆发 -> 奖励”链条

**追读力 E < 6 的章节，无论自然度分数多低，都需要重写。**

### 5. 平台留存力 Platform Retention（R）【v6 新增】

**这本书是否能让平台分发来的读者留下来？**

平台留存力不是玄学，也不是承诺爆款。它把番茄中文网公开可见的创作侧数据和工作流相连：书名/简介负责吸量，前三章负责初始读完，章节钩子负责追读，长线资产负责稳定留存。

- 低留存的病：书名简介承诺与正文不一致，第一章只解释设定，前三章没有兑现类型快感，8-10 万字前主线失焦。
- 高留存的特征：读者点进来后能立刻确认“这就是我想看的书”，每章都有新信息，关键节点有足够兑现，长线不靠注水拖延。
- 派生律：
  - 书名、简介、标签、第一章必须承诺一致。
  - 第一章必须兑现一个可见类型快感。
  - 前三章必须完成主角、金手指/核心能力、核心冲突、首个钩子。
  - 前 10 章必须有爽点梯度，不能连续铺垫。
  - 8-10 万字前必须形成可验证的首秀准备段：主线清晰、期待未断、爽点已兑现一轮。

## 三、L0 参数表（全包唯一阈值源）

以下参数是五维模型的可执行投影。所有文档引用阈值时，必须指向本表；脚本里的数值即本表的实现，不得另设。

| 维度 | 指标 | 阈值 | 含义 |
|------|------|------|------|
| 突发性 | 长段占比（≥55 字段占比） | <8% 判风险 | 长段太少 = 句长太均匀 |
| 突发性 | 节奏风险 rhythm_risk | >55 判风险 | 段长集中、句长规整 |
| 困惑度 | 语言模式风险 pattern_risk | >45 判风险 | 高概率词链 / 句首重复 |
| 信息熵 | 纹理风险 texture_risk | >45 判风险 | 设定词密集、私人纹理不足 |
| 综合 | 综合风险 total_risk | >35 优先修 | 整章机器腔风险 |
| 综合 | 分段风险 | >38 优先改片段 | 局部机器腔风险 |
| 篇幅 | 章节字数 | 2000-3000 中文字 | 各风格可在 L1 中细化 |
| 追读力 | 信息增量密度 | <0.8/500字 判风险 | 注水（与脚本一致） |
| 追读力 | 章末钩子强度 | C/D 级判风险 | 追读力不足（S=10,A=8,B=6,C=3,D=1） |
| 追读力 | 主角主动性 | 得分 <2 判风险 | 主角被动（主动动词×0.5 < 2 → 动词数 < 4） |
| 追读力 | 爽点密度 | 得分 <3 判风险 | 无爽点（冲突词×1.5 < 3 → 词数 < 2） |
| 追读力 | 综合追读力 E | <6 判风险 | 整体追读力不足 |
| 平台留存力 | 承诺一致性 | 书名/简介/第一章不一致判风险 | 吸量后留不住 |
| 平台留存力 | 黄金三章兑现 | 三章内未出现核心快感判风险 | 初始读完弱 |
| 平台留存力 | 前10章爽点梯度 | 连续3章纯铺垫判风险 | 追读曲线易断 |
| 平台留存力 | 8-10万字首秀准备 | 主线失焦或期待断档判风险 | 推荐验证弱 |
| 质量治理 | 低质风险 P0 | 主线不清、因果断裂、连续水化，或 total_risk >70 且 E<4 | 暂停正文，回前提/卷纲/细纲重做 |
| 质量治理 | 低质风险 P1 | total_risk >55，或信息密度不足且 E<6 | 必须重写结构或章节 |
| 质量治理 | 低质风险 P2 | 35 < total_risk <=55，或重复/POV/弱钩子问题 | 优先局部重写并复扫 |
| 质量治理 | 低质风险 P3 | 禁词、模板词或轻度模式化 | 定稿前清理 |
| 质量治理 | 分段预警阈值 | 分段扫描阈值 45：rhythm/pattern/texture/stats >=45；整章 55：total_risk >55 进入 P1 | 分段扫描比整章阈值更敏感，用于提前定位局部风险 |

注：长段阈值 55 字、长段占比 8%、综合 35、分段 38 均对应 `naturalness_scan.py`、`naturalness_batch_scan.py`、`naturalness_segment_queue.py` 的现行实现，作为事实源。追读力各项阈值与 `naturalness_scan.py` 中的实现完全一致。

## 四、风格实例入库门槛

L2 风格实例必须满足两个条件方可入库：

1. 通过 L1 创作律全部检验（不得含一级禁词、不得违反对白律等）。
2. 声明自己的五维指纹（在五个维度上偏哪一端）。

不满足的风格范例，可作为"原型存档"保留，但不进入生成管线。

## 五、系统架构

```text
L0  理论内核（本文件：五维模型 + 参数表 + 平台留存指标）
 │  派生（不可反向覆盖）
L1  创作律（references/creation-laws.md：段落律/对白律/用词律/POV律……）
 │  实例化
L2  类型与风格库
 │   ├── references/fangxiang-style.md（方想风格）
 │   ├── references/tianmo-style.md（天魔风格，已规范化）
 │   └── references/fanqie-style.md（番茄爆款风格）
 │  开书落地
L3  开书策略（references/platform-growth-model.md + story-craft/opening-strategy.md）
 │  章级生产
L4  章级追读引擎（satisfaction-engine.md + hook-cheatsheet.md + naturalness_scan.py）
 │  长线承接
L5  长线资产管理（世界观/伏笔/卷间规划/对标分析）
 │  质量闸门
L6  质量治理（naturalness-workflow.md + quality-governance.md + scripts/naturalness_*.py）
 │  发布复盘
L7  数据复盘（项目追踪文件：数据复盘/读者期待/低质风险）
```

铁律：下游不得与上游冲突。L2 风格不能违反 L1 律，L1 律必须能回溯 L0。L3-L7 只能把上游规则落到开书、写章、长线、质检和复盘，不能反向覆盖 L0-L2。

注意：L4 和 L5 的检测功能已内嵌于 `naturalness_scan.py` 中，不再设置独立脚本文件。
