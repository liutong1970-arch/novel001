# /wenxin — 文心·中文小说写作生产线

> 出自《文心雕龙》："夫文心者，言为文之用心也。"

## 定位

文心是一条完整的中文小说写作流水线，覆盖从选题到成品的全流程。内置去AI味引擎、三维编织写作法、多Agent协作审稿系统。

## 流水线总览

```
Phase 0: 选题 → Phase 1: 设定 → Phase 2: 大纲 → Phase 3: 正文 → Phase 4: 去AI味 → Phase 5: 审稿 → Phase 6: 输出
```

## 使用方式

```
/wenxin                    # 查看流水线状态和可用命令
/wenxin 新建 [书名]        # 创建新项目
/wenxin 选题               # 选题分析
/wenxin 设定               # 构建设定
/wenxin 大纲               # 设计大纲
/wenxin 写第X章            # 写指定章节（完整流程）
/wenxin 续写               # 继续写下一章
/wenxin 去AI味 [文本]      # 去除AI写作痕迹
/wenxin 审稿               # 多维度审查
/wenxin 状态               # 查看当前进度
/wenxin 搜索素材 [关键词]  # 搜索网络素材补充扩写
```

## 核心流程：写一章的完整步骤

**当你输入 `/wenxin 写第X章` 时，必须严格执行以下流程：**

### Step 1: 加载上下文
1. 读取本章细纲（`大纲/细纲/五星细纲/` 中对应文件）
2. 读取前章正文最后500字
3. 读取本章出场角色的卡片（`设定/角色/`）
4. 读取 `追踪/伏笔.md` 中已埋未揭的伏笔
5. 读取 `追踪/上下文.md` 中的当前写作位置

### Step 2: 三维编织写作

**每章必须包含5个场景，每个场景约500字，共2000-2500字。**

每个场景按三维编织法展开：

```
入场（1-2句）：在哪 + 干什么
  ↓
展开子事件：每个子事件在同一段落内交织三个维度
  ├── 发生：1-2句叙事，具体细节
  ├── 感知：感官细节，至少换一个感官
  └── 反应：身体反应 + 具体动作
  ↓
收尾：钩子或情绪定格
```

**关键规则**：
- 三个维度必须交织在同一段落，不能分开写成三段
- 每段 ≤ 60字
- 每段一句话
- 段落之间不留空行
- 对话独立成段
- 不用对话标签（"他说道""她回答道"）
- 连续描写段不超过3段
- 不切换视角

### Step 3: 去AI味检查

**写完后必须执行去AI味检查，这是硬性要求，不可跳过。**

#### 第一层：qu-ai-wei 51类模式检查

检查以下9大类AI腔模式（详见 `references/qu-ai-wei.md`）：

| 类别 | 检查内容 |
|------|---------|
| A. 内容模式 | 空洞拔高、背景套话、模糊归因 |
| B. 语言模式 | 高频词堆叠、名词化、机械并列 |
| B+. 逻辑连接 | 连接词空转、逻辑关系虚化 |
| C. 修辞模式 | 成语/排比模板化 |
| D. 交流模式 | 客服腔、谄媚腔 |
| E. 填充与模糊 | 冗余短语、模糊限定 |
| F. 翻译腔 | 英文句骨残留 |
| G. 篇章节奏 | 句长均质化 |
| I. 幻觉与格式 | 表格滥用、Markdown残留 |

#### 第二层：小说专用规则检查（详见 `references/novel-specific.md`）

| 检查项 | 标准 |
|--------|------|
| 对话书面化 | 加语气词，按角色区分 |
| 结尾升华 | 禁止哲理总结，以动作/对话/悬念结尾 |
| 禁词扫描 | 见下方禁词表 |

#### 第三层：过度修正三问

每处改写前验证：
1. 该表达在真人中文写作中常见吗？
2. 它在原文中是否承担特定功能？
3. 改写后是否与原文档位一致？

### Step 4: 字数验证

用 Python `len()` 计算中文字符数，目标 2000-2500字。

**字数不足时的两轮扩写流程：**

**第1轮：网络素材扩写**（详见 `references/material-search.md`）
1. 分析缺口场景，提取关键词
2. 使用 Scrapling / WebFetch / WebSearch 搜索素材
3. 筛选感官细节，融入三维编织格式
4. 重新验证字数

**第2轮：细纲分析 + 场景扩写**
1. 重新读取本章细纲，分析剧情要点
2. 读取前章正文和追踪文件，寻找衔接点
3. 新增1-2个符合情节逻辑的场景
4. 场景按三维编织法展开
5. 重新验证字数

**扩写原则**：✅ 符合细纲/角色/逻辑 ❌ 不添加无关情节

### Step 5: 更新追踪文件

写完后更新：
- `追踪/伏笔.md`：新埋/揭示伏笔
- `追踪/时间线.md`：新事件
- `追踪/角色状态.md`：角色状态变化
- `追踪/上下文.md`：当前进度

### Step 6: 确认完成

输出：
```
✅ 第X章完成
- 字数：XXXX字
- 去AI味：通过/需修改
- 伏笔操作：埋X个/揭X个
- 下一章：第X+1章
```

## 禁词表（必须检查）

### 最毒模式（6星）
"不是A，而是B" → 直接写B

### 一级禁词（立即替换）
- 模态：仿佛、犹如、宛若、如同、一丝、一抹
- 动作：深吸一口气、缓缓、不禁、微微、轻轻、淡淡
- 表情：眼中闪过、嘴角勾起、眉头微皱、瞳孔微缩
- 心理：心中一动、心头一震、心下了然、心中暗道
- 判断：不容置疑、显而易见、毫无疑问、不可否认
- 描写：坚定、闪烁着光芒、狡黠、深邃、凛冽

### 二级禁词（频繁时替换）
- 突然、好像、瞬间
- "他终于明白……"
- 连续3个以上排比段

### 破折号
尽量不用破折号（——），用逗号或句号代替。

## 角色设定

### 文心写手（narrative-writer）
- 模型：sonnet
- 职责：正文创作、三维编织、格式合规
- 最高优先级：字数合规

### 文心架构师（story-architect）
- 模型：opus
- 职责：大纲设计、世界观、结构审查

### 文心角色师（character-designer）
- 模型：sonnet
- 职责：角色设计、关系图谱、语言画像

### 文心检查员（consistency-checker）
- 模型：haiku
- 职责：grep-first 一致性检查

### 文心润色师（deai-editor）
- 模型：sonnet
- 职责：去AI味、qu-ai-wei 51类模式、小说专用规则

### 文心素材员（material-searcher）
- 模型：sonnet
- 职责：网络素材搜索、场景扩写、字数补充

## 质量标准

| 维度 | 标准 | 检查方式 |
|------|------|---------|
| 字数 | 2000-2500字/章 | Python len() |
| 段落 | ≤ 60字/段 | 逐段检查 |

## 重要提醒

**文心写作时必须遵守的铁律：**

1. **每章必须走完整流程**（加载上下文→三维编织→去AI味→字数验证→更新追踪）
2. **去AI味检查不可跳过**，这是硬性要求
3. **字数在2000-2500字为合格**，低于2000字必须补充
4. **字数不足时自动触发素材搜索扩写**
5. **不用破折号等AI常用语**
6. **逐章写作，写完一章确认达标后再写下一章**
7. **不贪快，不赶进度，质量第一**
