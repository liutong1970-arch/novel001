# 爽点工程体系（L4 追读引擎）

## 核心原则

1. **爽点是网文的生命线。** 没有爽点的小说，文笔再好也没人追读。
2. **爽点 = 期待 + 压抑 + 爆发 + 奖励。** 四个环节缺一不可。
3. **压抑不能超过爆发的 1/3。** 读者不是来受虐的。
4. **延迟满足比即时满足更爽。** 期待感越久，爆发越爽。

## 爽点类型

### 爽点结构完整性检查【v6】

每个有效爽点必须能填出四环节：期待、压抑、爆发、奖励。

```text
期待：读者提前知道主角想要什么或要证明什么。
压抑：阻力、误判、羞辱、资源差让读者产生不平。
爆发：主角用行动、能力或选择扭转局面。
奖励：地位、资源、关系、信息或情绪得到可见回报。
```

缺“奖励”的爽点不算完整；只有“系统提示奖励”但没有角色处境变化，也不算完整。

### 1. 打脸【服务:追读力】

**核心机制**：有人轻视/侮辱主角 → 主角展示实力/智慧 → 对方后悔/震惊。

**标准流程**：
1. 铺垫：配角轻视主角（"就凭你？""你懂什么？"）。
2. 压抑：配角继续挑衅，主角保持沉默或低调回应。
3. 爆发：主角展示实力/智慧，对方被打脸。
4. 奖励：旁观者震惊/佩服，主角获得地位/资源/好感。

**失败信号**：
- 没有铺垫就直接打脸（读者无感）。
- 打压时间太长（读者烦躁）。
- 打脸后没有奖励（读者不爽）。

### 2. 升级【服务:追读力】

**核心机制**：主角获得新能力/新资源 → 解决之前解决不了的问题。

**标准流程**：
1. 展示困境：主角遇到了当前实力解决不了的问题。
2. 获取途径：主角通过某种方式获得新能力/资源（修炼/奇遇/交易）。
3. 验证：新能力/资源解决了之前的问题。
4. 炫耀：旁观者震惊，主角获得认可。

**失败信号**：
- 升级过程太顺利（无趣）。
- 升级后没有立即使用（浪费期待）。
- 升级幅度太大（战力崩坏）。

### 3. 反杀【服务:追读力】

**核心机制**：主角处于劣势 → 利用智慧/环境/隐藏能力逆转 → 击败对手。

**标准流程**：
1. 劣势：主角明显弱于对手，形势危急。
2. 观察：主角寻找对手的弱点/环境的可利用之处。
3. 逆转：利用弱点/环境扭转局势。
4. 终结：彻底击败对手。

**失败信号**：
- 逆转没有铺垫（机械降神）。
- 对手太蠢（无成就感）。
- 逆转后没有代价（无紧张感）。

### 4. 被认可【服务:追读力】

**核心机制**：主角默默做事 → 有人（权威/重要角色）注意到 → 给予认可。

**标准流程**：
1. 隐藏：主角有能力但不展示。
2. 质疑：有人质疑主角的能力/价值。
3. 展示：主角在关键时刻展示能力。
4. 认可：权威/重要角色给予认可（资源/地位/好感）。

**失败信号**：
- 认可来得太容易（廉价）。
- 认可者不重要（无意义）。
- 认可后没有后续（浪费）。

### 5. 捡漏【服务:追读力】

**核心机制**：别人看不上的东西 → 主角发现其真正价值 → 获得巨大收益。

**标准流程**：
1. 忽视：众人忽视某物（"这有什么用？""破烂而已"）。
2. 发现：主角通过专业知识/特殊能力发现其价值。
3. 获取：以极低代价获得该物。
4. 验证：该物的价值被证实（解决重大问题/获得巨大收益）。

**失败信号**：
- 所有人都瞎了只有主角聪明（无趣）。
- 捡到的东西没有实际用途（浪费）。
- 没有验证过程（读者不信）。

### 6. 扮猪吃虎【服务:追读力】

**核心机制**：主角隐藏实力 → 被轻视/挑衅 → 展露实力 → 震惊全场。

**标准流程**：
1. 伪装：主角隐藏实力，表现得普通/弱小。
2. 挑衅：对手轻视主角，挑衅/攻击。
3. 展露：主角不再隐藏，展露真实实力。
4. 震惊：全场震惊，对手后悔/恐惧。

**失败信号**：
- 隐藏理由不合理（读者出戏）。
- 展露方式太刻意（"其实我是……"）。
- 展露后没有后续影响（浪费）。

## 爽点节奏

### 单章节奏

每章至少 1 个爽点时刻。推荐结构：

```text
场景1（50%）：铺垫 + 压抑
场景2（30%）：爆发 + 奖励
场景3（20%）：新悬念/钩子
```

### 多章节奏

| 章节范围 | 爽点类型 | 强度 | 说明 |
|---------|---------|------|------|
| 第1-3章 | 小爽点（捡漏/小打脸） | 低 | 建立期待，让读者留下来 |
| 第4-6章 | 中爽点（打脸/被认可） | 中 | 巩固追读习惯 |
| 第7-10章 | 大爽点（升级/反杀） | 高 | 第一次高潮，建立口碑 |
| 每3-5章 | 中爽点 | 中 | 维持追读率 |
| 每10-15章 | 超大爽点 | 极高 | 卷高潮，冲击推荐位 |

### 压抑-爆发比例

- **日常章**：压抑 1:爆发 2（轻松向）。
- **冲突章**：压抑 1:爆发 3（爽文向）。
- **低谷章**：压抑 2:爆发 1（但必须有爆发，不能纯压抑）。
- **绝对禁止**：连续 3 章以上纯压抑无爆发。

## 期待感管理

### 三阶段法则

1. **埋期待**：提前告知读者主角将要做某事/获得某物/面对某人。
2. **延迟满足**：不要立即兑现，让读者等待。
3. **充分兑现**：兑现时要充分、彻底、有冲击力。

### 延迟时长

| 期待类型 | 建议延迟 | 说明 |
|---------|---------|------|
| 小期待（日常） | 1-2 章 | 读者耐心有限 |
| 中期待（冲突） | 3-5 章 | 积累足够的压抑感 |
| 大期待（主线） | 10-20 章 | 长线布局，但中途要有小兑现 |

### 失败信号

- 埋了期待但忘了兑现（读者背叛感）。
- 延迟太久没有兑现（读者流失）。
- 兑现不充分（读者失望）。

## 题材专属爽点【v6】

| 题材 | 专属爽点 | 黄金三章要求 |
| --- | --- | --- |
| 都市异能 | 扮猪吃虎、规则收费、官方认可、危机反杀 | 第1章露出异能规则，第3章完成一次小兑现 |
| 系统流 | 任务奖励、面板升级、惩罚规避、隐藏成就 | 第1章绑定系统，第2章试用，第3章兑现奖励 |
| 重生 | 信息差打脸、避坑、提前布局、旧敌后悔 | 第1章确认重生优势，前三章改变一个旧结局 |
| 玄幻 | 境界突破、资源争夺、宗门认可、越级反杀 | 前三章露出修炼体系和第一次资源冲突 |
| 末世 | 囤货、避险、基地规则、怪物反杀 | 第1章危机倒计时，前10章完成生存闭环 |
| 直播/弹幕 | 弹幕反应、在线打脸、热度暴涨、观众站队 | 第1章出现互动反馈，第3章形成传播钩子 |

## 与信息密度的关系

爽点和信息密度是正相关的：

- 每个爽点都是一个信息增量（新事件/新能力/新关系）。
- 信息密度低的章节，通常也是爽点不足的章节。
- 两者可以一起检测：`scripts/naturalness_scan.py` 内嵌扫描信息增量和爽点。

## 与 POV 一致性的关系

爽点效果依赖 POV 一致性：

- 打脸/反杀的冲击力来自主角 POV 的体验。
- POV 漂移会削弱爽点效果（读者不知道应该代入谁）。
- 爽点场景中，POV 角色必须是亲历者。

## 与章末钩子的关系

爽点和章末钩子是互补的：

- 爽点在前，钩子在后。
- 大爽点之后的章末钩子最强（读者爽完了，想知道接下来发生什么）。
- 小爽点之后也需要钩子（读者刚爽完，需要新的期待）。
