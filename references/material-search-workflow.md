# 网络素材搜索与爬取工作流

> 素材不足、设定密集但私人纹理不足、场景偏短或用户明确要求真实细节时，必须搜索或复用素材；素材充足时不强制打断写章流程。
> 本文件已合并原 `material-search.md` 的内容，作为唯一素材搜索规范。
> 段长和字数引用 L0 参数表和 L1 长短交错律。

## 一、为什么需要素材触发机制

1. **用户明确要求**："字数不够不能幻想，要进行深度网络搜索，搜集素材补充"
2. **自然度质检需要**：机器腔最大的问题是"泛化描写"，真实素材提供具体细节，能补足私人纹理和现场取舍（L0 信息熵）
3. **感官丰富度**：自己想出来的细节往往是"老一套"，搜索能发现新鲜的感官组合
4. **跨学科知识**：食虫植物的捕食机制、搪瓷杯的制作工艺、老街的青砖质感——这些细节需要查证

## 二、触发条件

- 章节字数 < 2000 字（L0 参数表）
- 某个场景明显偏短（< 300 字）
- 自然度扫描提示 texture_risk 高，且设定词密集、私人纹理不足
- 场景涉及真实材质、职业、历史、地理、医疗、战斗等需要查证的信息
- 用户明确要求不能幻想、必须搜索或补充真实细节

## 三、搜索优先级

```text
1. web_search 关键词搜索 → 获取方向和关键词
2. httpx 爬取百科页面 → 获取具体感官细节
3. references/scraped-sensory-details.md → 已爬取的素材库
4. ❌ 不允许凭空编造
```

### 搜索技巧

1. **加"描写""素材""细节"等词**：搜"老人描写素材"比搜"老人"更有效
2. **加平台名**：搜"知乎 环境描写"能搜到写作经验帖
3. **搜百科**：搜"食虫植物 维基百科"能搜到科学细节
4. **搜具体材质**：搜"搪瓷杯 掉漆 黑铁皮"比搜"旧杯子"更具体

## 五、搜索工具

### 第一优先：WebSearch + httpx

```python
import httpx, re

resp = httpx.get(
    'https://zh.wikipedia.org/wiki/目标页面',
    timeout=15,
    headers={'User-Agent': 'Mozilla/5.0'}
)
text = resp.text

paras = re.findall(r'<p>(.*?)</p>', text, re.DOTALL)
results = []
for p in paras:
    clean = re.sub(r'<[^>]+>', '', p).strip()
    if len(clean) > 15 and len(clean) < 300:
        if any(w in clean for w in ['关键词1', '关键词2', '关键词3']):
            results.append(clean[:150])

for r in results[:10]:
    print(r)
    print()
```

### 备用方案

1. **Scrapling**：[Scrapling](https://github.com/D4Vinci/Scrapling) 自适应 Web 抓取框架（需 headless 浏览器）
2. **WebFetch**：直接获取指定 URL 的内容
3. **本地素材库**：使用 `scraped-sensory-details.md`

### 常用爬取目标

| 目标 | URL 模式 | 特点 |
|------|---------|------|
| 维基百科中文 | `zh.wikipedia.org/wiki/页面名` | 科学细节准确，适合植物/动物/材料 |
| 百度百科 | `baike.baidu.com/item/词条名` | 中文覆盖广，但 HTML 结构复杂 |
| 什么值得买 | `post.smzdm.com/p/文章ID` | 老物件评测，有具体感官细节 |
| 知乎专栏 | `zhuanlan.zhihu.com/p/文章ID` | 写作经验、场景描写技巧 |
| 科普中国 | `kepuchina.cn/wiki/...` | 科学类素材，权威性高 |

### 注意事项

- 百度百科 HTML 结构复杂，`<p>` 标签可能不包含正文，需要尝试其他选择器
- 维基百科中文是最可靠的爬取目标，HTML 结构稳定

## 六、素材筛选原则

1. **真实性**：优先选择有科学/历史依据的描写
2. **感官丰富**：包含视觉、听觉、嗅觉、触觉中至少两种
3. **符合时代**：素材需符合小说的时代背景
4. **拟人化**：避免模板化描写，优先补具体物件、身体反应和角色误判（L0 信息熵）

## 七、素材融入方法

1. **提取感官细节**：从搜索结果中提取具体的感官描写（非原文照搬）
2. **转换为三维编织格式**：融入发生 + 感知 + 反应
3. **融入原有情节**：不破坏节奏，不添加无关情节
4. **保持段长参差**：遵循 L1 长短交错律，短段为骨架，穿插长段
5. **融入后再次自然度复扫**：新内容可能引入段长规整、设定密度和高危词

## 八、已爬取素材库

每次爬取的素材应保存到 `references/scraped-sensory-details.md`，供后续章节复用。

### 素材记录格式

## 九、扩写流程（字数不足时）

```text
1. 分析缺口：哪些场景偏短？缺什么感官？
2. 提取关键词：从场景描述中提取 3-5 个搜索词
3. web_search 搜索：获取方向和页面 URL
4. httpx 爬取：从百科/科普页面提取具体细节
5. 筛选素材：选择符合时代、感官丰富、非模板化的细节
6. 融入正文：转换为三维编织格式，遵循 L1 长短交错律
7. 自然度复扫：扫描新内容是否引入禁词、段长规整或设定密度过高
8. 字数验证：确认达到 2000-3000 字（L0 参数表）
```

## 十、扩写原则

- ✅ 符合细纲要求
- ✅ 符合角色性格
- ✅ 符合情节逻辑
- ✅ 不破坏原有节奏
- ✅ 增加感官细节
- ✅ 推进情节或深化角色
- ❌ 不添加无关情节
- ❌ 不破坏人物设定
- ❌ 不与前后章矛盾

## 十一、实战案例

### 案例 1：盆栽恐怖化扩写

**需求**：第 9 章盆栽描写需要具体细节

**爬取**：维基百科捕蝇草页面
**提取素材**：
