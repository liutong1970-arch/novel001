# 世界观与势力设计（L5）

世界观不是百科。v6 的世界观只服务三件事：制造冲突、制造升级、制造长期期待。

## 一、力量体系

力量体系必须回答：

- 主角当前能做什么。
- 主角暂时不能做什么。
- 下一次升级能解决什么旧问题。
- 升级会带来什么新代价。

模板：

```markdown
体系名称：
能力来源：
等级阶梯：
当前上限：
升级资源：
升级代价：
克制关系：
失控风险：
```

红线：

- 等级只有名字，没有差异。
- 升级只加数值，不改变解决问题的方法。
- 主角每次升级都无代价。

## 二、势力结构

势力必须承担剧情功能，不只是背景名词。

```markdown
势力名称：
掌控资源：
公开目标：
隐藏目标：
与主角关系：
能给主角的压力：
能给主角的奖励：
首次出场章节：
预计升级/反转章节：
```

每个主要势力至少承担一种功能：

- 阻力：限制主角行动。
- 奖励：提供资源、身份、线索。
- 镜像：展示主角可能变成的样子。
- 反转：表面盟友，实际危险；表面敌人，实际可合作。

## 三、地图与区域

地图只写会被剧情使用的地方。

每个区域必须有：

- 一个可见规则。
- 一个资源点。
- 一个危险点。
- 一个会被回收的细节。

禁止：

- 一次性介绍多国、多宗门、多体系。
- 用地理说明替代主角行动。
- 区域换了，但冲突和爽点完全一样。

## 四、规则一致性

新增规则前检查：

- 是否解决当前剧情问题。
- 是否制造后续问题。
- 是否与旧规则冲突。
- 是否需要写入伏笔或角色状态。

如果新规则只为了让主角赢，判定为机械降神风险。

## 五、世界观审查表

```markdown
## 世界观审查

本章新增设定：
它服务的冲突：
它带来的奖励：
它带来的代价：
是否与旧规则冲突：
是否需要追踪：
```

通过标准：新增设定必须让主角更难、更强或更想继续行动。三者都没有，就删。
