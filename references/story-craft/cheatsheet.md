# Story-craft 速查表（wenxin-v6 更新）

用于快速决定当前问题该修哪里。优先级：平台承诺 > 故事发动机 > 细纲场景 > 追读质量 > 正文执行 > 低质风险 > 自然度质检。

## 问题归因

| 读感问题 | 先检查 | 如果失败 | 修法 |
| --- | --- | --- | --- |
| AI 味重 | 场景是否有目标/阻力/转变 | 没有 | 修细纲，把抽象指令改成角色选择 |
| AI 味重 | 正文是否堆比喻/模板反应 | 有 | 修正文，白描动作和代价 |
| 点开就走 | 书名/简介承诺是否在第一章兑现 | 没有 | 修开书承诺和第一章事件 |
| 开篇不抓人 | 是否有催化事件和亲近点 | 没有 | 增加角色主动牺牲或冒险 |
| 爽点不爽 | 是否有压抑、误判、代价 | 没有 | 先加阻力，再兑现 |
| 对白说明腔 | 对话双方是否各有目的 | 没有 | 重设冲突，不让台词承担百科 |
| 章节像流水账 | 每场价值是否改变 | 没有 | 合并或重写无转变场景 |
| 人物扁 | 是否有欲望/需求/弱点 | 没有 | 补角色卡，再写行动 |
| 追读率低 | 章末钩子弱、信息密度低、主角被动 | 没有 | 加钩子、提密度、加主动行动 |
| 配角脸谱化 | 反应模式单一，缺乏身份/性格区分 | 有 | 为不同角色设计不同反应方式 |
| POV 混乱 | 视角切换，写了角色不知道的事 | 有 | 严格保持 POV 一致性 |
| 重复段落 | 章内或章间重复率 >80% | 有 | 删除或重写重复段落 |
| 内容水 | 每 500 字是否有有效信息增量 | 没有 | 删水段，补新事件/新冲突/新关系 |
| 逻辑割裂 | 事件之间是否有因果 | 没有 | 回细纲重排因果链 |

## 细纲改写公式

差的细纲句：

```text
展示灾难压迫感，体现主角震惊。
```

改成：

```text
主角想把灾难引离人群，切断求救频道，失去最后一次被定位救援的机会；黑火没有追最近的人，反而追她的血迹，说明它有识别目标。
```

## 场景最低通过线（wenxin-v6 更新：2-3 个深场景）

每场必须能填出：

```text
目标：
阻力：
行动：
意外反应：
开头价值：
结尾价值：
代价：
钩子：
爽点类型：
```

缺三项以上，先别写正文。

## 第一章通过线

| 项目 | 合格标准 |
| --- | --- |
| 类型承诺 | 读者知道这是什么类型的爽文/悬疑/异能故事 |
| 亲近点 | 关键人物做了一个带代价的选择 |
| 世界规则 | 规则通过行动结果展示，不靠百科说明 |
| 主角/关键人物 | 有身份，也有弱点、习惯、判断方式 |
| 章末钩子 | 新问题比旧问题更大 |

## 正文自然度质检优先级

1. 先跑 `scripts/naturalness_scan.py`。
2. 改结构统计特征：段长、主语重复、信息推进过匀。
3. 增加私人纹理：偏见、误判、身体不适、追责压力。
4. 改设定说明：让设定通过错误数据、失效工具、误判后果露出。
5. 最后才删泛化比喻、主题总结、模板反应。
6. 保留毛边：口语、犹豫、短句可以是人味，不要全部磨平。

## 追读质量检查（wenxin-v6 新增）

1. **信息密度**：每 500 字至少有一个信息增量。
2. **爽点**：每章至少 1 个爽点时刻。
3. **章末钩子**：结尾是否让读者想点下一章。
4. **主角主动性**：每章至少 1 次主动选择和行动。
5. **POV 一致性**：是否只写了 POV 角色知道的事。
6. **重复段落**：章内和章间重复率是否 >80%。
7. **配角反应变体**：不同角色是否有不同的恐惧/情绪表现方式。

## 低质风险检查

1. **水化**：删去 30% 是否仍不影响剧情。
2. **模板堆砌**：爽点是否只有公式，没有铺垫和代价。
3. **逻辑割裂**：新能力、新人物、新证据是否突然出现。
4. **批量感**：同类桥段、同类恐惧、同类章末是否连续复用。
5. **处理原则**：P0/P1 回细纲，P2 局部重写，P3 定稿前清理。

## 判断句

- 没有目标和阻力，是细纲问题。
- 有目标和阻力但读起来假，是正文问题。
- 人物只会震惊，是角色卡问题。
- 对白只在解释世界观，是场景冲突问题。
- 比喻很多但行动很少，是写作执行问题。
- 章末像环境描写，是钩子问题。
- 每 500 字没有信息增量，是节奏问题。
- 主角一直在被动应对，是角色塑造问题。
- 多个角色用同一种恐惧表现方式，是配角塑造问题。
- 书名简介承诺没有在第一章兑现，是平台承诺问题。
- 删掉大段文字不影响剧情，是内容水化问题。
