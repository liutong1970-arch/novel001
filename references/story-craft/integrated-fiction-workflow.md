# 统一小说创作流程（wenxin-v6）

v6 把小说生产拆成四层闭环：项目级承诺、卷级长线、章节级追读、发布后复盘。每层必须串行通过，不允许同时大改多个层级。

## 一、项目级：先定承诺

写正文前必须完成：

- 题材和目标读者。
- 书名、简介、标签承诺。
- 主角欲望、核心阻力、失败代价。
- 黄金三章规划。
- 前 10 章追读梯度。

必读：

- `platform-growth-model.md`
- `opening-strategy.md`
- `benchmark-analysis.md`

如果书名简介承诺和第一章内容不一致，暂停正文。

## 二、卷级：再定长线

每卷必须明确：

```text
卷核心欲望
卷核心阻力
卷核心爽点
卷中点反转
卷高潮兑现
卷末钩子
下一卷承诺
```

必读：

- `world-building.md`
- `foreshadowing.md`
- `long-term-planning.md`

如果连续 3 章只换地点不换冲突，回到卷级规划。

## 三、章节级：写章前闸门

每章先补齐：

```text
本章功能：
本章承诺：
当前风格：
POV角色：
角色目标：
阻力：
主动行动：
世界的意外反应：
价值转变：
情绪起点/终点/变化原因：
信息增量：
爽点类型：
伏笔新增/推进/回收：
章末钩子：
素材需求：
低质风险预判：
```

硬规则：

- 缺目标、阻力、主动行动，先修细纲。
- 缺信息增量，判定为注水风险。
- 缺爽点且不是明确过渡章，必须补兑现。
- 缺章末钩子，不能进入正文。
- POV 不明确，不能写正文。

## 四、正文执行

正文采用：

```text
发生 + 感知 + 反应
```

执行标准：

- 每章 2000-3000 中文字，按题材可微调。
- 每 500 字至少一个信息增量。
- 每章至少一次主角主动选择或行动。
- 每章至少一个爽点或明确过渡功能。
- 章末必须 C 级以上钩子，D 级禁止。
- 严格保持 POV 一致性。
- 感官和比喻只服务行动，不替代行动。
- 情绪值只进细纲和追踪表，正文用动作、身体反馈、选择和代价外化。
- 设定密集但私人纹理不足时，读取 `material-search-workflow.md` 和 `scraped-sensory-details.md` 补素材。

## 五、完稿质检

固定顺序：

1. 结构审稿：是否有不可逆变化。
2. 信息密度：是否注水。
3. 爽点检查：是否完成期待、压抑、爆发、奖励。
4. 钩子检查：章末是否能推动下一章。
5. POV 检查：是否写了角色不知道的事。
6. 重复检查：是否有重复段落或桥段。
7. 自然度质检：是否机器腔、过匀、模板化。
8. 情绪连续性：情绪变化是否有触发事件和外化动作。
9. 素材纹理：设定密集但私人纹理不足时，是否读取素材库补足具体细节。
10. 低质风险：是否水化、逻辑割裂、模板堆砌。
11. 追踪更新：伏笔、时间线、角色状态、读者期待、爽点、钩子、情绪追踪、低质风险。

前四项不通过，不进入语言润色。

## 六、修改归因

| 现象 | 优先归因 | 修改方向 |
| --- | --- | --- |
| 读者点进来就走 | 承诺错位 | 修书名/简介/第一章兑现 |
| 事件多但不想看 | 主角欲望弱 | 加目标、代价、阻力 |
| 爽点不爽 | 铺垫或奖励不足 | 补期待链条 |
| 追读低 | 钩子弱/信息少 | 加新问题和未兑现期待 |
| AI味重 | 抽象细纲 + 模板正文 | 改选择、代价、手迹 |
| 内容水 | 信息增量不足 | 删水段，补有效事件 |
| 长线散 | 伏笔和卷目标断 | 回到长线规划 |

## 七、发布后复盘

拿到读完率、追读、互动或流量变化后，只做一件事：把数据转化为修改动作。

```markdown
掉点章节：
疑似原因：
对应模块：
修改动作：
复查时间：
```

一次复盘最多 3 个动作。超过 3 个，优先处理最早掉点章节。
