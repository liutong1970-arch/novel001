# 场景设计（wenxin-v6 更新）

用于写章前的场景闸门、`/wenxin-v6 场景转折审查`。

## 场景最低标准

一个场景必须有：
- 角色目标。
- 阻力。
- 行动。
- 意外反应。
- 价值转变。
- 新信息或新代价。

如果没有价值转变，它只是段落，不是场景。

## 场景卡（wenxin-v6 更新）

```markdown

场景功能：

```

## 价值转变示例

| 价值轴 | 开头 | 结尾 |
| --- | --- | --- |
| 安全/危险 | 安全 | 危险 |
| 尊严/屈辱 | 保住脸面 | 当众受辱 |
| 真相/谎言 | 误信谎言 | 看见破绽 |
| 亲近/疏离 | 暂时合作 | 关系破裂 |
| 自信/怀疑 | 以为能赢 | 开始动摇 |

## 深场景节奏（wenxin-v6 更新：2-3 个深场景，不再是 5 个浅场景）

```text
场景1：进入问题，给出本章目标。（约占 40%）
场景2：冲突升级 + 爽点爆发 + 价值转变。（约占 40%）
场景3：章末钩子 + 推进下一章。（约占 20%）
```

**为什么从 5 个改为 2-3 个：**
- 5 个浅场景导致每个场景只有 360-520 字，没有一个场景能让读者沉浸。
- 2-3 个深场景让每个场景有完整的情绪周期（压抑→爆发→收获）。
- 番茄/起点读者需要的是深度沉浸，不是广度覆盖。

## 与三维编织合并

`scene-design` 管戏剧骨架，`wenxin` 三维编织管正文密度。

每个段落仍按：

```text
发生 + 感知 + 反应
```

但每个场景整体必须能回答：

```text
这场之前是什么状态？
这场之后什么东西变了？
谁为这个变化付了代价？
读者为什么要翻下一页？
```

## POV 一致性检查（wenxin-v6 新增）

每个场景必须标注 POV 角色。同一场景内不得切换 POV。

```markdown
## POV 检查

| 场景 | POV 角色 | 是否切换 | 备注 |
| --- | --- | --- | --- |
| 场景1 | 路天明 | 否 | |
| 场景2 | 路天明 | 否 | |
```

如果同一场景内切换了 POV，必须用 *** 或空行分隔，并标明新 POV。
