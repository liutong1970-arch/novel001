# 故事架构

用于 `/wenxin-v6 故事体检`、`/wenxin-v6 前提审查`、选题、大纲前诊断。

## 输入

- 一句话故事前提。
- 主角初设。
- 类型或目标读者。
- 已有大纲或章节梗概。

## 前提审查表

| 项 | 问题 | 合格标准 |
| --- | --- | --- |
| 主角 | 故事绑定的是谁？ | 一个最承受变化的人 |
| 欲望 | 他具体想得到什么？ | 可见、可验证、能失败 |
| 阻力 | 谁或什么阻止他？ | 阻力会主动升级 |
| 代价 | 失败会失去什么？ | 不是尴尬，而是生活结构受损 |
| 需求 | 他真正该学会什么？ | 与表层欲望有张力 |
| 价值 | 核心价值两极是什么？ | 如尊严/屈辱、真相/谎言 |
| 类型承诺 | 读者期待什么快感？ | 每卷都能兑现 |

## Truby 七步骨架

```text
弱点/需求：
欲望：
对手：
计划：
对决：
真实自我揭露：
新平衡：
```

检查：
- 弱点必须在故事开始就伤害主角生活。
- 欲望必须把主角推出舒适区。
- 对手必须攻击主角弱点，而不是只挡路。
- 计划必须会失败，迫使主角升级。
- 对决必须让主角付代价。
- 自我揭露不能靠别人说教。
- 新平衡要体现主角已经不同。

## McKee 价值命题

为故事指定核心价值对：

```text
核心价值：____ / ____
开局状态：主角处在____
中段反转：价值转向____
高潮选择：主角为了____放弃/承担____
结局状态：世界或主角稳定在____
```

## 常见病

- 前提只有设定，没有欲望。
- 主角只有遭遇，没有行动。
- 反派只坏，不构成价值对立。
- 主题写成口号，没有场景证明。
- 高潮靠突然出现的新信息解决。
- 主角成功了，但没有付出不可逆代价。
